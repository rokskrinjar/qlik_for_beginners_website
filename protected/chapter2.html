2.1 Understanding Data Loading in Qlik

Data loading is one of the most critical aspects of creating a successful Qlik application. Before you can visualize and analyze data, you need to load it into Qlik's associative engine. In this chapter, we'll explore how Qlik's data loading process works and the various methods for bringing data into your applications.

Qlik's approach to data loading is different from many other BI tools. Instead of connecting directly to source systems for every query, Qlik loads all the data into memory. This in-memory approach offers several advantages:

  • Performance - Data is available instantly for analysis without waiting for database queries
  • Associative engine - All data relationships are maintained, allowing for non-linear exploration
  • Data transformation - You can clean, reshape, and enhance your data during the loading process
  • Offline access - Once loaded, the data is available even without connection to source systems

2.2 Connecting to Data Sources

Qlik can connect to a wide variety of data sources, from simple files to complex databases. Here are the most common data sources used with Qlik:

Data Source Type Examples Best For
Flat Files CSV, Excel, TXT, QVD Simple data sources, extracted data, data exchange
Databases SQL Server, Oracle, MySQL, PostgreSQL Large datasets, transactional systems
Web Sources REST APIs, Web Files, HTML Tables External data, online services

1Connect to Excel Data

Let's practice connecting to a simple Excel file:

  1. Create a new Qlik Sense app
  2. Click "Add data"
  3. Select "Excel" from the data connections list
  4. Browse to your Excel file and select it
  5. You'll see a preview of the tables in the file
  6. Select the tables you wish to import
  7. Click "Add data" to load the selected tables

2.3 Using the Data Manager

The Data Manager is Qlik's visual interface for loading and preparing data. It's designed to be user-friendly, making it ideal for beginners. With the Data Manager, you can:

  • Connect to various data sources
  • Preview data before loading
  • Combine multiple tables
  • Create associations between tables
  • Profile your data to identify quality issues
  • Apply basic transformations without scripting
The Qlik Sense Data Manager interface showing data preview and associations

While the Data Manager is an excellent starting point, it has limitations for complex data models. As you advance in your Qlik journey, you'll likely need to use the Data Load Editor for more complex transformations and loads.

2.4 Introduction to the Data Load Editor

The Data Load Editor is where you'll write scripts to load and transform data. It gives you complete control over the data loading process, allowing for complex transformations, calculations, and data model design.

To access the Data Load Editor:

  1. Open your app
  2. Click "Prepare" in the navigation menu (or "Data manager" button)
  3. Click "Data load editor" button in the bottom right
The Data Load Editor showing a sample load script

The Data Load Editor consists of several key components:

  • Data connections panel - Lists available connections to data sources
  • Script editor - Where you write your load script
  • Toolbar - Contains functions for editing, debugging, and running the script
  • Tabs - Helps organize your script into logical sections

2.5 Script Basics

The Qlik load script uses a specific syntax that combines SQL-like statements with Qlik's own scripting language. Here are some fundamental concepts:

Basic Script Structure

// Comments start with double slashes

// Connect to a data source
LIB CONNECT TO 'MyConnection';

// Load data from a table
LOAD 
    Field1,
    Field2,
    Field3 as RenamedField
FROM [lib://MyConnection/MyTable.csv]
(txt, codepage is 1252, embedded labels, delimiter is ',');

// Create a calculated field in another table
LOAD
    ID,
    Amount,
    Date,
    Amount * 1.1 as AmountWithTax
FROM [lib://MyConnection/Sales.xlsx]
(ooxml, embedded labels, table is Sheet1);

// Store the data for faster loading next time
STORE Sales INTO [lib://MyConnection/Sales.qvd];

Key Script Commands

  • LOAD - Loads data from a file or generates data
  • SELECT - Retrieves data from a database using SQL
  • STORE - Saves loaded tables to QVD files for faster future loading
  • JOIN - Combines tables based on matching fields
  • CONCATENATE - Appends one table to another
  • MAP - Creates a mapping table for lookups
  • RENAME FIELD - Changes a field name
  • DROP TABLE - Removes a table from memory
  • LET - Assigns a value to a variable

2.6 Creating Associations Between Tables

One of Qlik's most powerful features is its associative model, which maintains relationships between data tables. Tables are automatically associated when they share field names, but you can also explicitly define associations.

Automatic Associations

When tables share fields with the same name, Qlik automatically creates associations between them. For example, if both your Orders table and Products table have a field called "Product_ID", they will be associated.

Manual Associations

If your fields have different names but should be linked, you can rename fields during the load process:

LOAD
    Customer,
    Order_ID,
    Product as Product_ID, // Renamed to match the Products table
    Amount,
    Date
FROM [lib://Data/Orders.xlsx]
(ooxml, embedded labels, table is Orders);
The Data Model Viewer showing associations between tables

2.7 Data Loading Best Practices

As you develop your Qlik skills, keep these best practices in mind:

  • Use QVD files for intermediate storage to improve load performance
  • Minimize the number of fields you load to only what's needed
  • Apply transformations during the load rather than in visualizations
  • Use proper naming conventions for fields and tables
  • Add comments to document your script
  • Organize your script into logical sections using tabs
  • Include error handling for more robust scripts
  • Test incrementally rather than writing a long script before testing

Practice Exercise: Create a new Qlik app and load data from at least two different sources (such as an Excel file and a CSV file). Make sure they share at least one field to create an association, then view the data model to confirm they're linked correctly.
